Firebase 동기화 설정 가이드

행사 설정을 모든 PC/브라우저에서 공유하기 위한 5분 세팅

1. Firebase 프로젝트 만들기

  1. Firebase Console 접속 (구글 로그인)
  2. 프로젝트 추가 → 이름은 자유 (예: excel-promo) → 만들기
  3. Google Analytics는 비활성화해도 됩니다.

2. Firestore Database 활성화

  1. 좌측 메뉴 → 빌드 → Firestore Database
  2. 데이터베이스 만들기 → 위치는 asia-northeast3 (서울) 또는 asia-northeast1 (도쿄) 추천
  3. 모드는 프로덕션 모드 선택 → 사용 설정

3. 보안 규칙 설정 (가장 중요)

Firestore Database → 규칙 탭에서 아래로 교체 후 게시:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /configs/{docId} {
      allow read, write: if true;
    }
  }
}
⚠️ 위 규칙은 이 Firebase 프로젝트의 web app config(API key 등)를 아는 사람이면 누구나 읽고 쓸 수 있습니다. 개인용으로 본인만 사용한다면 충분히 안전합니다 (config를 외부에 공유하지 않으면 됨). 더 엄격한 보호가 필요하면 익명 인증 + 인증된 사용자만 쓰기 규칙으로 바꾸세요.

4. 웹 앱 등록 후 config 받기

  1. 좌측 메뉴 → 프로젝트 설정 (톱니바퀴)
  2. 일반 탭 하단 → 내 앱 → </> 웹 아이콘 클릭
  3. 닉네임(예: excel-promo-web) 입력 → 앱 등록
  4. 화면에 보이는 firebaseConfig 객체 부분을 복사: 
    const firebaseConfig = {
  apiKey: "AIza...",
  authDomain: "myproj.firebaseapp.com",
  projectId: "myproj",
  storageBucket: "myproj.appspot.com",
  messagingSenderId: "...",
  appId: "1:..."
};
    JSON 형태로 (key를 따옴표로 감싸서) 변환해 붙여넣어야 합니다: 
    {
  "apiKey": "AIza...",
  "authDomain": "myproj.firebaseapp.com",
  "projectId": "myproj",
  "appId": "1:..."
}

5. 엑셀 처리기에 붙여넣기

  1. 엑셀 행사 처리기 → 메뉴1 → 5. 클라우드 동기화
  2. "Firebase config 입력" 펼치기 → 위 JSON을 그대로 붙여넣기 → 설정 저장
  3. ↑ 클라우드에 올리기 클릭 → 현재 행사 설정이 Firestore에 저장됨
  4. 다른 PC에서 같은 페이지 → 같은 JSON 붙여넣고 → ↓ 클라우드에서 내려받기
  5. (옵션) 변경 시 자동으로 올리기를 체크하면 메뉴1에서 행사 추가/삭제할 때마다 1.5초 뒤 자동 동기화됨

요금

Firestore 무료 등급은 매일 50,000회 읽기/20,000회 쓰기/1GB 저장까지 무료입니다. 이 정도 사용량이면 평생 무료로 쓸 수 있어요.
